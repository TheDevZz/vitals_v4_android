# 血压分析跨进程服务示例

这个示例展示了如何在Android应用中创建一个跨进程服务来调用BloodPressureAnalyzer.analyze方法。

## 文件结构

### AIDL接口文件
- `src/main/aidl/com/vitals/example/IBloodPressureAnalyzerService.aidl` - 定义跨进程通信接口，使用统一的ParcelableLiveVitalsSampledData

### Java类
- `src/main/java/com/vitals/example/BloodPressureMeasureResult.java` - 实现Parcelable的血压测量结果类

### Kotlin类
- `src/main/java/com/vitals/example/BloodPressureAnalyzerService.kt` - 血压分析服务实现，直接使用统一的数据类型
- `src/main/java/com/vitals/example/ResultActivity.kt` - 修改后的结果页面，直接使用统一的数据类型

### 配置文件
- `src/main/AndroidManifest.xml` - 注册血压分析服务

## 服务特性

1. **独立进程**: 服务运行在独立的进程 `:blood_pressure_service` 中
2. **AIDL通信**: 使用AIDL进行跨进程通信
3. **Parcelable序列化**: 支持复杂数据结构的跨进程传递
4. **异步处理**: 在后台线程中进行血压分析，避免阻塞UI
5. **统一数据类型**: 所有模块使用相同的数据类型，无需类型转换

## 使用方法

1. 在ResultActivity中，设置 `useService = true`
2. 服务会自动绑定并开始血压分析
3. 分析结果会通过AIDL回调返回并显示在UI上

## 数据流程

1. `ResultActivity` 获取 `VitalsSampledData`
2. 调用 `genParcelableSampledData()` 转换为 `ParcelableLiveVitalsSampledData`
3. 通过AIDL直接传递给 `BloodPressureAnalyzerService`
4. 服务直接调用 `BloodPressureAnalyzer.analyze()` 进行分析
5. 将结果转换为 `BloodPressureMeasureResult` 并返回
6. `ResultActivity` 接收结果并显示

## 关键实现细节

### 统一数据类型
通过修改native_bp模块的包名，实现了数据类型的完全统一：
- 所有模块都使用 `com.vitals.sdk.parcel.ParcelableLiveVitalsSampledData`
- BloodPressureAnalyzer直接使用统一的数据类型
- AIDL接口使用统一的数据类型
- 无需任何类型转换，代码更简洁

### 错误处理
- 服务连接异常处理
- 远程调用异常处理
- 血压分析异常处理

### 生命周期管理
- 服务绑定和解绑
- Activity销毁时自动解绑服务

## 注意事项

- 所有模块使用统一的数据类型 `com.vitals.sdk.parcel.ParcelableLiveVitalsSampledData`
- 服务运行在独立进程中，需要处理进程间通信的异常
- 记得在Activity销毁时解绑服务
- 代码更简洁，无需类型转换

## 测试方法

1. 运行应用
2. 进入血压测量流程
3. 在ResultActivity中查看是否使用服务进行分析
4. 检查日志输出确认服务正常工作
5. 验证血压分析结果是否正确显示

## 日志标签

- `BloodPressureAnalyzerService`: 服务相关日志
- `ResultActivity`: Activity相关日志
- `BPAnalyzer`: BloodPressureAnalyzer内部日志
